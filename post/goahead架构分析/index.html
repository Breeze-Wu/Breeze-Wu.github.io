goahead架构分析

by dgwu
2023-07-08

摘要

本文旨在梳理总结goahead源码(4.1.3)实现原理，解析大致的函数栈调用路线，学习其架构设计思路。文中错误处欢迎指正交流。

1、goahead是什么？

GoAhead Web Server是一款基于C语言编写的轻量级和高性能的Web服务器，它由微型嵌入式Web服务器制造商Embedthis Software研发并持续维护。

GoAhead的主要特点是占用很少的内存，CPU消耗低廉，同时功能却非常强大，支持CGI、SSI、摘要认证等标准Web特性。因此，GoAhead广泛应用于追求高性能、低成本的嵌入式系统和各种较为简单Web服务的场景。

1.1 goahead支持的功能

HTTP服务器引擎 ：

  • 完全支持HTTP/1.1
  • 支持IPv4和IPv6
  • 单线程、基于事件的服务器
  • 请求路由和重写
  • 支持分块和流水线请求
  • 错误和访问日志记录（常见日志格式）
  • 沙盒资源限制
  • 会话状态存储

性能

  • 请求吞吐量（> 10,000个请求/秒）
  • 基于事件的核心，有效处理多个请求
  • 小内存占用：115K代码

模块化

  • 条件编译选项
  • 支持CGI、JavaScript模板、文件上传和SSL的处理程序
  • 可配置的请求路由
  • 可扩展/可替换的授权

安全性

  • 安全套接字层（SSL）
  • 基本、摘要和表单身份验证
  • 基于路由的授权
  • 沙盒限制
  • 访问和访问违规日志记录
  • 安全的可移植运行时，以防止内存泄漏和缓冲区溢出

动态内容

  • 服务器端JavaScript
  • CGI/1.1
  • GoActions - 内存中的URL操作
  • 自定义C处理程序

2、GoAhead代码流程概述

main()

  |

  |--websOpen()

  |                |--初始化handle函数并与route.txt中的路由绑定，注意“/”路由默认绑定了file这个handle函数 所有路由信息都存在route全局数组变量中

  |

  |--(main websOpen)

  |                |--websActionOpen()

  |                |--websFileOpen()

  |                |--websCgiOpen()

  |                |                |--websDefineHandler

  |                |                |--上面三个opention都会调用websDefineHandler方法，创建WebsHander结构量并存到hander哈希结构中

  |                |--websLoad()

  |                |                |--解析route.txt配置文件，调用websAddRoute绑定url与websHander结构，并新建WesRouter结构，所有WesRouter存到route数组中
  
  |                |                |--websAddRoute()

  |
  
  |--websListen()

  |                |--创建服务器侦听socket，IP地址默认为1127.0.0.0:80

  |                |--每个socket都新建了一个WebsSocket结构，并存放到socklist数组中

  |      

  |--websServiceEvents()

  |                |--循环监听socklist数组中的socket，将其加入select中并处理连接消息

  |

  |--(main websServiceEvents)
  
  |                |--socketSelect()

  |                |                |--轮询socketList[]

  |                |--socketProcess()

  |                |                |--socketDoEvent()

  |                                		|--如果有新的连接(来自listenfd)就调用socketAccept()

  |                                     |                  |        |--调用socketAlloc()初始化一个WebsSocket结构并存到socketList哈希结构中，供select事件侦听使用

  |                                     |                  |        |--调用accept接收连接并将连接新socket赋值给WebsSocket.sock

  |                                     |                  |        |--websAccept()
  |                                     |                  |        |    |--websAccept负责创建Webs结构，用来维护本次连接的相关状态信息，所有Webs结构量放在webs全局数组结构中管理
    
  |                                     |                  |        |--调用socket_t sp->accept()回调函数去接收连接并将新socket放到WebsSocket结构、设置新连接的event处理函数为socketEvent

  |                                     |

  |                                     |--如果不是新的连接就查找socketList数组调用socket_t sp->handler()回调函数(实际是调用socketEvent)

  | 
  
  |--(main websServiceEvents socketProcess socketDoEvent socketEvent)
  
  |                |--readEvent()   处理读事件

  |                |                |--websRead()

  |                                		|--读取socket缓冲区数据
  
  |                |                |--websPump()

  |                                		|--用状态机解析http消息

  | 
  
  |--(main websServiceEvents socketProcess socketDoEvent socketEvent readEvent websPump)
  
  |                |--parseIncoming()

  |                |                |--parseFirstLine()

  |                |                |--parseHeaders()

  |                                		|--解析http消息

  |                |--websRouteRequest()

  |                |                |--根据http的url到route中查找路由并调用url对应的handle方法，handle方法在前面websOpen中统一注册到了handle这个全局哈希结构中
  
  |                |--processContent()

  |                |                |--处理put、cig、upload等http数据

  |                |--websRunRequest()

  |                |                |--执行route.handle.server，这里就进入只定义的handle方法中处理http数据，如json数据

  | 
  
  |--其他接口函数
  |                |--websSetStatus(wp, 200);
  |                |--websWriteHeaders(wp, -1, 0);
  |                |--websWriteEndHeaders(wp);
  |                |--websWrite(wp, "%s", str);
  |                |--websFlush(wp, 0);
  |                |--websDone(wp);  //关闭连接状态
--|

3、总结

goahead是一个轻量的单线程web服务器，其源码架构实现可分为三个阶段，首先解析route.txt，绑定url与handle回调函数，其次创建socket开启web服务侦听，接收客户端连接，最大连接数量为内核参数SOMAXCON(128)，最后使用select管理socket读写事件。

goahead的优势上面已经有列举，在使用过程中也发现其存在一些不足，首先是单线程，在上传大文件时，等等时间比较长，并且只能等待文件上传结束才能继续处理消息。其次是goahead服务端关闭socket，这导致系统中产生大量的IME_WAIT。
